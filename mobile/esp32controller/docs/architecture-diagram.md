## Key Design Decisions

### 1. **Discriminated Unions for Type Safety**
```typescript
// âœ… TypeScript knows result shape based on 'ok' field
type DeviceResponse = DeviceSuccess | DeviceError;
```

### 2. **Factory Pattern for Mock Support**
```typescript
// âœ… Single interface, multiple implementations
const client = createClient(useMock);
```

### 3. **Normalized Response Shape**
```typescript
// âœ… Always returns { ok, ...data } - no exceptions thrown
const result = await led(ip, 'toggle');
```

### 4. **Pre-Request Validation**
```typescript
// âœ… Catch errors before network request
if (!isValidIP(ip)) return { ok: false, error: '...', ... };
```

### 5. **Built-in Latency Tracking**
```typescript
// âœ… Every response includes timing info
{ latencyMs: Date.now() - startTime }
```

### 6. **User-Friendly Error Categories**
```typescript
// âœ… 6 distinct error types for targeted UX
errorType: 'timeout' | 'network' | 'http' | 'validation' | 'cancelled' | 'unknown'
```

## Security Considerations

1. **Input Validation**: All IPs and URLs validated before requests
2. **No Code Execution**: Plain text responses only
3. **Timeout Protection**: All requests have max timeout
4. **No Secrets**: IP addresses are user-configured, not stored
5. **Local Network**: ESP32 typically on same WiFi network
